# Ingest inbox

Drop-zone for raw sources before they get processed into the federated wiki.

## Workflow

1. **Любой контрибьютор** (координатор курса, ментор, студент, заинтересованный посторонний) кладёт сюда raw-материал через PR: CSV ретро, транскрипт митинга, экспорт чата, скрин, заметку.
2. **Координатор/агент с compute** забирает файл и:
   - предлагает каноничное место для raw,
   - создаёт `source-summary`-карту в `wiki/` или в sibling-view (`retros/`, `audits/`),
   - обновляет `wiki/index.md` и `wiki/log.md` (action: `ingest`).
3. После обработки `ingest/` снова пустая. Это очередь, не архив.

## Канонические места для raw (после обработки)

| Тип материала | Куда уходит raw | Публично? |
|---|---|---|
| Ретро (CSV студенческих ответов) | `retros/raw/<YYYY-MM-DD>-<slug>.csv` | **gitignore** |
| Federation-level событие (без course-owner) | `sources/<YYYY-MM-DD>-<slug>/raw/` | **gitignore** |
| Аудит-источник (репозиторий, сторонний документ) | `audits/raw/<YYYY-MM-DD>-<slug>.<ext>` | **gitignore** |
| Чат-экспорт, скрин, прочее | предлагается агентом при обработке | по обстоятельствам |

**Course-bound материалы сюда не идут.** Транскрипты митингов конкретного курса, peer-review CSV конкретного курса, scoring-таблицы конкретного курса — лежат **в репо самого курса** (например, `rs-tandem-dashboard/courses/<course-id>/meetings/`). В этот wiki-репо едут только federation-уровня обобщения.

Если каноничное место не очевидно — агент **спрашивает**, прежде чем переместить.

## Privacy / publish rules

Raw материалы, в которых есть **PII** (имена, GitHub-handles, прямые отзывы о людях по именам) — не идут в публичный коммит. Архитектурно:

- `retros/raw/`, `audits/raw/`, `sources/*/raw/` — **gitignore по умолчанию** (см. `.gitignore`). Это приватный архив координатора.
- В публичный коммит едут **только summary-карты** (`retros/<date>.md`, `audits/<date>.md`), которые написаны без имён и идентифицирующих деталей.
- Если raw-источник *уже публичный* (запись митинга на YouTube, открытый GitHub-репозиторий) — его можно положить в обычное место рядом с summary, без `raw/` подпапки. Имена в публично-доступных контекстах сохраняются.
- Если в raw есть смесь публичного и приватного (транскрипт открытого митинга + чат с гостями-студентами) — приватная часть в `raw/` (gitignore), публичная рядом с summary.

Анонимизация в summary:
- Студенты, гости, рекрутёры — без имён (роль, локация, контекст).
- Координаторы школы (head-координаторы курсов) — обычно с именами, они и так публичны.
- Менторы — без имён в негативных отзывах. В позитивных — на усмотрение координатора (по умолчанию: тоже без имён).

## Authority / merge model

- **PR в `ingest/`** — открыт всем. Raw валидируется на отсутствие явного PII перед мерджем.
- **PR в `wiki/`, `retros/`, `audits/`, `sources/`** — мерджит только координатор (по умолчанию — Воробей). Это гейт, который не даёт LLM-генерации одного контрибьютора попасть в публичные факты школы без проверки.

## Что **не** идёт через ingest

- Готовые `source-summary` или `entity` страницы — открываются прямо как PR в `wiki/` или sibling-view, минуя ingest.
- Course-bound артефакты — в репо самого курса.
- Маркетинг / лендинги / эссе — это другой контур (см. `vision.md` §1.3).
