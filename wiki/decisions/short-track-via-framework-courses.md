# Short Track 2026 — вторая часть проводится через RS School React/Angular курсы

## Решение

В 2026-м году **второй модуль EPAM × RS School Short Track заменён на полноценные RS School курсы по React или Angular**.

Конкретно:

1. Кандидат, успешно прошедший qualifying stage, **не идёт на отдельный second-stage Short Track**, а попадает в один из дедикейтед курсов:
   - **Short-track React-курс** (14 недель), курирует Dmitry Baranov: JavaScript → TypeScript → OOP → функциональные основы → React → Node.js → SDLC → AI.
   - **Общий Angular-курс** RS School (выделенного «short-track Angular» в 2026 нет).
2. **Lab-места распределяются по результатам этих курсов**, не по итогам qualifying-stage demo. Demo учитывается, но **не как основной сигнал** («the QS results will not be heavily considered», прямая речь Воробья на митинге 2026-04-30).
3. Региональные ограничения сохраняются: дедикейтед React-курс доступен резидентам Грузии, Казахстана, Кыргызстана, Армении, Узбекистана, Турции (= зона активного найма EPAM в 2026, см. `infrastructure/epam-hiring-locations.md`).
4. Решение применяется **к когорте 2026 года**. Продолжится ли в 2027-м — открытый вопрос (зависит от того, как сработает в 2026-м).

## Почему

**Контекст.** Старая модель Short Track в EPAM × RS School была: qualifying stage → второй модуль (короткий, заточенный под EPAM-интервью) → Lab → Junior. В 2026-м эта схема меняется.

**Аргументы в пользу замены:**

1. **Глубина подготовки**. 14-недельный курс под кураторством Dmitry Baranov даёт более крепкую базу под junior-интервью EPAM, чем сжатый второй модуль. Прямая цитата из митинга: «junior should know basics — алгоритмы, hash-maps, как делить программу на модули, SDLC» — 14-недельный формат это закрывает.

2. **Использование существующих сильных активов**. RS School уже умеет хорошо вести длинные курсы (Tandem, Angular). Делать ещё один параллельный «short-track-second» — дублирование. Логичнее канализировать поток через уже работающие курсы.

3. **AI-сдвиг в индустрии (см. `decisions/score-philosophy-after-ai.md`)**. EPAM в открытую говорит: «знай архитектуру, знай базу, знай AI как инструмент — не просто синтаксис фреймворка». Расширенный курс даёт время это закрыть; второй модуль short-track — нет.

4. **Сигнал «не QS demo, а курс»**. Один из эффектов решения — снижение веса qualifying-stage demo в финальном решении о Lab. Это больше соответствует реальности: на demo студенты часто перформят не своим обычным уровнем; курс с дневниками и ревью даёт более честную картину.

## Что это значит для координатора

Если ты координируешь курс, через который EPAM возьмёт студентов в Lab (короче — любой курс RS School после 2026-04-30):

- **Считай свой курс частью EPAM-pipeline**, явно или неявно. Это не «учим React», это «готовим к технической работе».
- **Дневник + peer-review + защита** — теперь не только внутренние артефакты курса, но и сигналы для EPAM-рекрутёра. Соответственно их форматирование/экспорт стоит закладывать с учётом «это будет смотреть внешний оценщик».
- **Финальная защита** имеет повышенный вес: формирует решение о Lab. См. `decisions/peer-review-precedes-defence.md` — peer-review как репетиция перед финальной защитой особенно актуален.
- **Координатор курса в зоне найма** (Грузия, Казахстан, Кыргызстан, Армения, Узбекистан, Турция, плюс Польша для Angular) сообщает студентам о пути в Lab явно, на kickoff. См. `decisions/lock-rules-at-kickoff.md` — путь к Lab может быть бенефитом курса, но он должен быть **явно зафиксирован** на kickoff, иначе создаст ощущение «правила меняются».

## Что это значит для студента

- **QS demo важно, но не единственно**. Если по demo результат скромный, а в курсе ты разберёшься глубоко — у тебя есть второй шанс через перформанс на курсе.
- **«Lab гарантирован после короткого курса» — миф**. Решение о Lab — комбинация **результатов курса + наличия открытой позиции в EPAM в твоём регионе**. Курс закрывает первое; второе — рынок.
- **Если ты не в зоне найма EPAM-2026** (UK, например, или Беларусь без особого случая) — курс полезен, но Lab прямой не светит. Альтернативы: общий React/Angular курс RS School + независимое подавание в EPAM по официальным каналам.

## Что не закрывает это решение

- **Запуск short-track 2027** — будет ли вторая часть снова через курсы или вернётся отдельный модуль. Решение нужно пересмотреть к концу 2026 на основе результата 2026-й когорты.
- **Что считается «успешным завершением курса» для целей Lab**. Сейчас формулировка размытая («результаты курса»). На kickoff такого курса нужно явно зафиксировать: «Lab-кандидаты — top-N по dashboard / выполнили полную программу / получили сертификат». Иначе студент не понимает, к чему стремиться.
- **Метрика, по которой EPAM смотрит на курс**. Сейчас сигналы — финальная защита + защита перед EPAM-командой. Возможно стоит добавить explicit-checkpoint типа «code review с EPAM-инженером», чтобы качество подготовки имело чёткую точку контроля.
